# Outline — quantification-llm

## Thèse centrale
La quantification des LLM n'est pas une troncature de précision : c'est une **bataille contre les valeurs aberrantes** (*outliers*). Une poignée de canaux d'activation et de poids saillants concentre l'essentiel de l'erreur ; chaque méthode sérieuse (LLM.int8, SmoothQuant, GPTQ, AWQ, rotations QuaRot/SpinQuant, QAT, ternaire natif) est une stratégie différente pour **protéger, déplacer ou dissoudre ces outliers** — et sous 4 bits par poids, la qualité tombe d'une falaise que seules ces stratégies repoussent.

## Angle / différenciation
Deep dive de `ia-embarquee` (suite a explicitement listée au backlog). ia-embarquee traitait la quantification comme *un levier parmi d'autres* du budget mémoire de l'appareil. Ici on lit **la mécanique serrée** : les algorithmes eux-mêmes, la méta-évaluation, les formats matériels 2025-2026. Famille « compression », distincte des 3 derniers dossiers « systèmes distribués » (melange-experts, kv-cache, désagrégation).

## Plan (8 sections)
1. **Pourquoi quantifier** — decode *memory-bound* : le débit est plafonné par la bande passante mémoire, pas le calcul ; réduire les octets par poids = accélérer directement (renvoi kv-cache/ia-embarquee). Poids vs activations vs KV-cache comme trois budgets distincts. W-only vs W+A.
2. **L'échelle des formats** — ce que « N bits » veut dire : quantification uniforme affine (scale + zero-point), symétrique/asymétrique, granularité (per-tensor / per-channel / per-group). Formats : INT8, INT4, FP8 (E4M3/E5M2), NF4, ternaire. Le décodage se fait en FP16, on ne gagne que la mémoire+bande passante (sauf W8A8).
3. **La découverte fondatrice : les outliers** — LLM.int8() (Dettmers 2022) : au-delà de ~6,7B params, des *emergent features* — quelques dimensions d'activation à magnitude extrême — cassent la quantification naïve. Décomposition à précision mixte. SmoothQuant (Xiao 2023) : migrer la difficulté activation→poids par une mise à l'échelle par canal (α), car les poids se quantifient mieux que les activations.
4. **PTQ sans réentraînement** — la famille dominante en pratique. GPTQ (Frantar 2023) : compensation d'erreur ordre par ordre via l'inverse de la Hessienne (OBQ passé à l'échelle). AWQ (Lin, MLSys 2024 best paper) : 1 % de poids saillants portent l'essentiel de l'erreur ; les protéger par mise à l'échelle *activation-aware* sans backprop → préserve la généralisation. GGUF k-quants (llama.cpp) : quantification par blocs à bits mixtes (Q4_K_M), le format de facto de l'edge. AQLM/QuIP# : quantification additive / codebooks pour 2 bits.
5. **Les rotations** — QuaRot (Ashkboos, NeurIPS 2024) : une rotation orthogonale (Hadamard aléatoire) fusionnée dans les poids rend le réseau *computationnellement équivalent* mais **sans outliers** → 4-bit end-to-end (poids+activations+KV). SpinQuant (Liu 2024, ICLR 2025) : *apprendre* la rotation (optimisation de Cayley) bat Hadamard fixe — gap à FP16 réduit à 2,9 pts sur LLaMA-2 7B en W4A4KV4. L'insight : l'outlier n'est pas intrinsèque, c'est un artefact de base ; on change de base.
6. **QAT — quand on peut réentraîner** — Straight-Through Estimator (gradient à travers l'arrondi). LLM-QAT, EfficientQAT. Apple Foundation ~2-bit par QAT. Le cas limite : **ternaire natif** BitNet b1.58 (Wang 2024) — entraîner en 1,58 bit dès le départ, pas quantifier après coup ; élimine les multiplications. QAT récupère la qualité mais coûte du calcul d'entraînement.
7. **Méta-évaluation : mesurer sans se mentir** — la perplexité masque les régressions (une PPL quasi-intacte cache un effondrement en raisonnement/code). La falaise sous 4 bits. Sensibilité par tâche (code et maths plus fragiles que le chat). W8A8 (SmoothQuant) vs W4A16 (AWQ/GPTQ) vs KV-quant : trois compromis pour trois goulots. Le piège de la sur-adaptation au jeu de calibration.
8. **Trajectoires 2026-2028** — QAT par défaut quand le budget d'entraînement le permet ; formats matériels natifs : **MXFP4** (OCP, bloc 32, exposant partagé E8M0) et **NVFP4** (NVIDIA Blackwell, bloc 16, double échelle FP8+FP32, ×3,5 vs FP16, <1 % de perte) exécutés nativement par les tensor cores ; co-conception format×matériel×méthode (MR-GPTQ) ; la frontière 1-bit ; quantification du KV-cache comme poste dominant à contexte long.

## Schémas (7) + régions cliquables
- **SCHEMA-01 — L'échelle des bits** (landscape) : échelle FP16→FP8→INT8→INT4→ternaire, mémoire par poids, facteur ×, zone « falaise <4 bits ». Régions : (a) FP16/BF16 baseline, (b) INT8/FP8 palier sûr, (c) INT4 falaise, (d) ternaire natif.
- **SCHEMA-02 — Anatomie d'une quantification affine** (landscape) : distribution des poids, grille de niveaux, scale+zero-point, granularité per-tensor/channel/group. Régions : (a) scale & zero-point, (b) symétrique vs asymétrique, (c) per-channel vs per-group, (d) déquantification à l'inférence.
- **SCHEMA-03 — Le problème des outliers** (landscape) : histogramme d'activations avec quelques canaux extrêmes ; migration SmoothQuant (α) poids↔activation ; décomposition mixte LLM.int8. Régions : (a) emergent features, (b) pourquoi ça casse INT8, (c) SmoothQuant migration, (d) décomposition mixte.
- **SCHEMA-04 — Taxonomie des méthodes PTQ** (square) : arbre W-only (GPTQ, AWQ, GGUF, AQLM/QuIP#) vs W+A (SmoothQuant, LLM.int8). Régions : (a) GPTQ Hessienne, (b) AWQ activation-aware, (c) GGUF k-quant blocs, (d) codebooks 2-bit.
- **SCHEMA-05 — La rotation qui dissout les outliers** (landscape) : avant/après rotation orthogonale, distribution aplatie, fusion dans les poids ; Hadamard fixe vs rotation apprise. Régions : (a) base d'origine avec outliers, (b) rotation Hadamard, (c) équivalence computationnelle, (d) rotation apprise SpinQuant.
- **SCHEMA-06 — PTQ vs QAT : coût contre qualité** (2×2 / courbe) : axe coût de calibration (minutes → jours GPU) × qualité récupérée ; la falaise sous 4 bits repoussée par QAT et rotations. Régions : (a) PTQ rapide, (b) QAT coûteux, (c) falaise 4-bit, (d) ternaire natif hors-échelle.
- **SCHEMA-07 — Matrice méthodes × dimensions** (landscape) : lignes = méthodes (LLM.int8, SmoothQuant, GPTQ, AWQ, QuaRot, SpinQuant, GGUF, BitNet, NVFP4) ; colonnes = bits, W/A/KV, réentraînement, support matériel, qualité. Régions : (a) colonne réentraînement, (b) colonne matériel, (c) ligne rotations, (d) ligne formats natifs.
