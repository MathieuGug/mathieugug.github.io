7 attaques · trajectoires sur les 3 couches

Chaque attaque traverse les 3 couches du gruyère. Pour chacune, on lit où elle est arrêtée (✓), où elle passe (✗), et où la défense est redondante ou rare.

Prompt injection

  1. N1 · Préventif ✗ PASSE

    Tests ne couvrent que les patterns connus

    • Templates testés en isolation
    • Garak / PromptInject scannent
    • mais les injections nouvelles passent
    • Garak
    • DeepEval
  2. N2 · Détectif ✓ ARRÊTE

    Anomaly detection + 👎 spikes en prod

    • Distribution-shift sur outputs
    • User feedback signale du bizarre
    • Détection en 2-12h
    • Lakera
    • Sentry
    • Datadog
  3. N3 · Correctif ⊘ REDONDANT

    Manual review filet de sécurité

    • Sample hebdomadaire détecte
    • les injections évidentes
    • enrichit le corpus d’attaques
    • LangSmith

Hallucination

  1. N1 · Préventif ✓ ARRÊTE

    µ-3 Golden regression + Output validation

    • Tasks figées détectent
    • le drift sur cas connus
    • JSON schema force le format
    • HaluEval
    • TruthfulQA
    • FActScore
  2. N2 · Détectif ⊘ REDONDANT

    A/B verrait NPS chute

    • + user feedback "l’IA invente"
    • détection rapide
    • déjà stoppé en amont
    • Statsig
  3. N3 · Correctif ⊘ REDONDANT

    Manual review filet sécurité

    • → feed nouvelles golden tasks
    • boucle vertueuse
    • playbook → gruyère → playbook
    • LangSmith

Tool misuse

  1. N1 · Préventif ✓ ARRÊTE

    µ-2 Tool schema validation typée

    • JSON Schema / OpenAPI / Zod
    • arguments typés et bornés
    • rejette le mauvais appel
    • JSON Schema
    • OpenAPI
    • Zod
  2. N2 · Détectif ⊘ REDONDANT

    5xx spikes en prod

    • + "ça plante" feedback
    • rapide mais après-coup
    • C1 est plus efficace
    • Sentry
  3. N3 · Correctif ⊘ RARE

    Pas le bon outil ici

    • Validation programmatique gagne
    • haut la main sur tool calls
    • review humaine = overkill

Context saturation

  1. N1 · Préventif ✗ PASSE

    Trajectory bench partiel

    • Tests typiques 2-3 turns
    • pas 12+ comme en prod
    • ⚠ trou large
    • τ-bench

  2. N2 · Détectif ✗ PASSE

    • Signal indirect

    • difficile à diagnostiquer
    • PostHog
  3. N3 · Correctif ✓ ARRÊTE

    Manual review observe

    • l’agent perdre le fil
    • sur 12+ turns
    • signal direct et lisible
    • Honeycomb
    • LangSmith

Reward hacking

  1. N1 · Préventif ✗ PASSE

    Graders deterministic contournés

    • explicites et les évite
    • Goodhart’s law en action
    • BBH

  2. N2 · Détectif ✗ PASSE

    Métrique monte, but réel baisse

    • Proxy decoupling
    • A/B trompé par sur-optim
    • invisible en aggregate
    • Statsig
  3. N3 · Correctif ✓ ARRÊTE

    Human studies + red team manual

    • Détectent les patterns subtils
    • inter-rater agreement
    • gold standard
    • MACHIAVELLI
    • RewardBench

Spec mismatch

  1. N1 · Préventif ✗ PASSE

    Schema OK pour format,

  2. N2 · Détectif ✓ ARRÊTE

    A/B révèle re-prompts/abandons

    • Les users redemandent autrement

    • Statsig

    Review catégorise les types

    • LangSmith

Latency drift

  1. N1 · Préventif ✗ PASSE

    Perf bench 1 task isolée

    • Pas trajectoire 12 tool calls
    • ⚠ TROU LARGE
    • fix : ajouter trajectory bench
    • k6
    • Promptfoo
  2. N2 · Détectif ✗ PASSE

    A/B utilise success_rate

    • pas latency comme métrique 1°
    • user tolère 1-2s, signale > 4s
    • fix : latency obligatoire
    • Statsig
  3. N3 · Correctif ✗ PASSE

    Review note la précision,

    • pas le timing
    • sample 20/sem ≪ M traces
    • fix : checklist timing
    • LangSmith

Issues possibles

Les 7 attaques produisent l'une des deux issues, selon que le gruyère a couvert ou non.

Agent fiable

Attaque arrêtée par au moins une couche · 6 attaques sur 7 stoppées dans notre modèle.

Incident

Trous alignés sur les 3 couches · l'attaque traverse sans être interceptée. Dans notre modèle, c'est latency drift.

« L'empilement n'est pas une garantie ; c'est une réduction de la probabilité qu'une attaque atteigne la prod. La maintenance (Phase 3) re-pondère les trous au fur et à mesure que le modèle évolue. »
