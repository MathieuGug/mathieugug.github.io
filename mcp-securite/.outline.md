# Outline — sécurité MCP

**Slug**: `mcp-securite`
**Date**: 2026-05-20
**Type**: étude (deep dive sur `mcp-plateforme`)
**Cible mots**: ~5500-7000

## Thèse

La surface d'attaque de Model Context Protocol n'est pas un bug à patcher mais une géométrie à comprendre. Les dix vecteurs d'attaque s'organisent en quatre familles — tool layer, prompt layer, transport, identité — qui appellent dix patterns défensifs spécifiques. **Le contrat de confiance se construit avant l'exécution, par signature et tagging à l'install — pas après, par audit en runtime.**

## Plan (8 sections)

1. **Synthèse exécutive** — 5 bullets : géométrie de la surface, les 4 familles, l'asymétrie attaque/défense MCP, le pivot Sigstore + signature, la matrice défensive 10×10
2. **La géométrie de la surface MCP** — anatomie d'un appel MCP, où chaque étape ouvre un trust boundary. [SCHEMA-01 : anatomie d'un appel + 6 trust boundaries]
3. **Famille A — Tool poisoning et description spoofing** — TPA Invariant Labs 2025, descriptions invisibles, shadowing d'outils, line-jumping. [SCHEMA-02 : variants TPA]
4. **Famille B — Prompt injection cross-document et data exfiltration** — la lecture LLM comme exécution, exfil via tool calls, l'attaque GitHub via README, l'attaque WhatsApp Asana ; rug pull. [SCHEMA-03 : la chaîne d'exfil]
5. **Famille C — Cross-server confusion, namespace collisions, shadowing** — quand plusieurs serveurs MCP coexistent, le LLM s'embrouille. [SCHEMA-04 : grille des 10 vecteurs × 4 familles]
6. **Famille D — Transport, OAuth, supply chain** — DCR pitfalls, token passthrough, session hijack, NPM/PyPI typosquatting MCP, sécurité du transport SSE/Streamable HTTP. [SCHEMA-05 : flow OAuth MCP avec pitfalls annotés]
7. **La matrice défensive — dix patterns** — Sigstore signing, tool tagging, sandbox isolé, allowlist namespace, human-in-the-loop pour writes, OTel audit log, OAuth scoping serré, content provenance, rate limiting, refresh token rotation. [SCHEMA-06 : matrice 10 vecteurs × 10 patterns]
8. **Roadmap 12 mois** — spec MCP automne 2026 (signature Sigstore, isolation, audit log standardisé), AAIF révision, EU AI Act art. 15 sur la cybersec des systèmes IA, montée des registries MCP signés. [SCHEMA-07 : timeline]
