# Outline — MCP, le HTTP des agents

## Thèse

==Le Model Context Protocol gagne par effet de réseau, pas par sa rigueur technique : un protocole simple, mal fini, mais adopté par tout le monde, en train de devenir la couche d'interopérabilité par défaut entre LLMs et outils — au prix d'une surface d'attaque encore largement non gouvernée.==

## Plan (7 sections, ~5 500 mots, 6 schémas)

1. **Synthèse** — pourquoi MCP est le HTTP des agents : 970× de croissance en 18 mois, donation Linux Foundation, 78 % des équipes IA enterprise en production.
2. **Genèse : du tool-use ad-hoc au protocole partagé** — fragmentation 2023-2024 (function calling propriétaire OpenAI / Anthropic / Google), publication MCP novembre 2024, bascule mars 2025 quand OpenAI rejoint, donation décembre 2025.
3. **Architecture : client, serveur, transports** — JSON-RPC 2.0, primitives (tools / resources / prompts / sampling), stdio vs Streamable HTTP, composition. — Schéma 1 (architecture protocole) + Schéma 3 (composition multi-server)
4. **L'écosystème en 2026** — 17 000 serveurs indexés, 300+ clients, registre officiel + concurrents (MCP.so), distribution Python/TS/Go. — Schéma 2 (timeline d'adoption)
5. **La surface de risque** — tool poisoning, indirect prompt injection, cross-server contamination, MCP sampling, supply chain. Aucun chain-of-trust sur les descriptions à runtime. OWASP MCP Top 10. — Schéma 4 (taxonomie des attaques)
6. **MCP, function calling, OpenAPI, A2A : qui mange quoi ?** — pas de concurrence frontale, layering différent. UTCP comme rappel que MCP n'est pas définitif. — Schéma 5 (matrice comparative)
7. **Trajectoire 2026-2027** — commodification de la couche outil, gouvernance via AAIF, prochaine couche (registries d'identité, signature de descriptions, sandboxing). — Schéma 6 (trajectoire)

## Schémas (6)

1. **Anatomie du protocole MCP** (architecture, landscape) — host / client / serveur, message JSON-RPC, primitives. Régions : `host`, `client`, `server`, `transport`, `primitives`.
2. **Effet de réseau : 18 mois d'adoption** (timeline, landscape) — nov 2024 lancement, mars 2025 OpenAI, été 2025 Microsoft, sept 2025 registre officiel, déc 2025 donation Linux Foundation, Q1 2026 78 % enterprise. Régions : `launch`, `openai-pivot`, `microsoft`, `donation`, `enterprise`.
3. **Composition multi-server : un agent, quatre intégrations** (architecture, landscape) — agent au centre, 4 serveurs (GitHub, Slack, Postgres, filesystem), bus JSON-RPC. Régions : `agent`, `mcp-github`, `mcp-slack`, `mcp-postgres`, `mcp-fs`.
4. **Surface d'attaque MCP** (taxonomie, square) — 4 vecteurs : tool poisoning, indirect prompt injection (data plane), cross-server contamination, supply chain (paquet npm/pypi). Régions : `tool-poisoning`, `indirect-injection`, `cross-server`, `supply-chain`.
5. **Quatre couches d'intégration LLM ↔ outils** (matrice, landscape) — function calling / MCP / OpenAPI / A2A : axes "périmètre" (intra-app → cross-org) et "abstraction" (modèle → service). Régions : `function-calling`, `mcp`, `openapi`, `a2a`.
6. **Trajectoire 2026-2027** (timeline, landscape) — trois pistes : commodification (registres concurrents → consolidation), gouvernance (OWASP MCP Top 10, signature des descriptions), prochaine couche (sampling, sub-agents, A2A interop). Régions : `commodification`, `gouvernance`, `prochaine-couche`.
