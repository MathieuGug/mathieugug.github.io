# Outline — attention-parcimonieuse

## Thèse centrale
L'attention parcimonieuse a changé de nature en 2025 : longtemps optimisation greffée à l'inférence (motifs fixes, éviction gloutonne du cache), elle est devenue une **propriété entraînée** du modèle — le réseau apprend *pendant le pré-entraînement* quels tokens ignorer, avec des noyaux alignés sur le matériel. Cette bascule referme le trou de généralisation (OOD) qui condamnait l'éviction post-hoc : **la parcimonie native est à l'éviction ce que MLA est à GQA**.

Deep dive prolongeant `compression-kv-cache` (suite c « Native Sparse Attention »). Distinction porteuse : `compression-kv-cache` réduisait la *taille* d'un cache déjà calculé (partage de têtes, latent, quantification, éviction) ; ce dossier attaque l'autre moitié — **ne pas calculer** l'attention sur la plupart des tokens, décision remontée à l'entraînement.

## Plan (8 sections)

1. **Le mur quadratique** — l'attention dense coûte O(n²) : à long contexte, le *calcul* du prefill devient le goulot (distinct de la *mémoire* du KV-cache, objet de `kv-cache`/`compression-kv-cache`). La parcimonie = attaquer le calcul, pas seulement les octets. Ce que « parcimonieux » veut dire (chaque requête ne regarde que k≪n clés). [SCHEMA-01]
2. **Deux générations de parcimonie** — motifs fixes conçus à la main (Sparse Transformers strided/fixed 2019 ; Longformer fenêtre+global 2020 ; BigBird fenêtre+global+aléatoire, approximateur universel/Turing-complet). Puis parcimonie *dynamique/apprise*. L'axe structurant : *où* la décision de parcimonie est prise. [SCHEMA-02]
3. **La ligne de partage : post-hoc vs entraîné** — training-free (éviction du cache H2O/StreamingLLM/SnapKV ; parcimonie de prefill MInference, motifs A-shape/vertical-slash/block) vs trained-in. Le risque OOD de l'éviction gloutonne : le modèle n'a jamais vu ces trous à l'entraînement. Analogie MLA/GQA. [SCHEMA-03]
4. **NSA — l'anatomie des trois branches** — compression (blocs grossiers résumés), sélection (top-n blocs fins), fenêtre glissante (local) ; gate appris qui pondère ; alignement matériel (kernels GQA-aware, intensité arithmétique équilibrée) ; *natively trainable* (gradient à travers la sélection par blocs). 11× sur 64k, jusqu'à 1M tokens, qualité ≥ dense. ACL 2025 Best Paper. [SCHEMA-04]
5. **MoBA — l'attention comme mélange d'experts** — blocs = experts, routeur top-k par requête, principe « less structure » (pas de biais fenêtre/sink imposé) ; bascule sans couture full↔sparse ; déployé en production sur Kimi. NSA vs MoBA : trois branches fixes apprises vs routage MoE-like des blocs. [SCHEMA-05]
6. **La voie de l'adaptation : SeerAttention & le gate distillé** — pas besoin de pré-entraîner de zéro : distiller la parcimonie *intrinsèque* d'un modèle dense (AttnGate, self-distillation sur le 2D-maxpool des cartes d'attention, kernel block-sparse FlashAttention-compatible). Deux voies vers la parcimonie entraînée : from-scratch (NSA/MoBA) vs post-training (SeerAttention). SeerAttention-R pour le raisonnement long. [SCHEMA-06]
7. **La production : DeepSeek Sparse Attention (V3.2-Exp)** — le *lightning indexer* (fonction de similarité pondérée légère, entraînée sur 2,1 Md tokens à imiter la dense de V3.1-Terminus) + sélection fine top-k ; complexité cœur O(L²)→O(Lk) ; qualité quasi identique, −50 % prix API, support day-0 vLLM/SGLang. Passage d'artefact de recherche à brique de service. [SCHEMA-07]
8. **Trajectoires 2026-2028** — sparse par défaut au pré-entraînement ; co-conception sparse × MLA (attention = système mémoire couplé, pont `compression-kv-cache`) ; sparse × MoE (double parcimonie) ; théorie de la généralisation OOD de la sélection apprise ; obsolescence de Long Range Arena, besoin de bancs long-context reasoning. Renvois vers `kv-cache`, `melange-experts`, `economie-inference`. [SCHEMA-08 → non, on garde 7 schémas ; section textuelle]

## Schémas (7) + zones cliquables
- **S1 — Le mur quadratique** (landscape) : courbe coût attention vs longueur de séquence, dense O(n²) vs sparse O(n·k) ; encart prefill(calcul) vs decode(mémoire). Zones : `cout-dense`, `cout-sparse`, `point-bascule`, `prefill-vs-decode`.
- **S2 — Deux générations** (landscape, timeline/taxo) : frise motifs-fixes (Sparse T. → Longformer → BigBird) → parcimonie apprise (NSA/MoBA/Seer/DSA). Zones : `motifs-fixes`, `sparse-transformers`, `bigbird-theorie`, `bascule-apprise`.
- **S3 — La ligne de partage** (square 2×2) : axes = « décision de parcimonie prise à l'entraînement ↔ à l'inférence » × « granularité cache/token ↔ calcul attention » ; placer H2O, StreamingLLM, MInference, NSA, MoBA, SeerAttention, DSA. Zones : `quadrant-posthoc`, `quadrant-entraine`, `risque-ood`, `analogie-mla`.
- **S4 — NSA, trois branches** (landscape architecture) : requête → [branche compression | branche sélection top-n | branche fenêtre] → gate appris → sortie ; bandeau alignement matériel. Zones : `branche-compression`, `branche-selection`, `branche-fenetre`, `gate-appris`, `hardware-align`.
- **S5 — MoBA, l'attention-MoE** (landscape) : séquence découpée en blocs = experts ; routeur top-k par requête ; comparaison NSA (branches fixes) vs MoBA (routage). Zones : `blocs-experts`, `routeur-topk`, `less-structure`, `switch-full-sparse`.
- **S6 — Le gate distillé** (landscape pipeline) : modèle dense → cartes d'attention → 2D-maxpool (cible) → AttnGate appris → block-sparse kernel ; from-scratch vs post-training. Zones : `attn-maps`, `maxpool-cible`, `attngate`, `deux-voies`.
- **S7 — DSA en production** (landscape) : lightning indexer + top-k selection sur MLA ; courbe coût linéaire O(Lk) ; timeline adoption (V3.1→V3.2-Exp→vLLM/SGLang day-0, −50 % API). Zones : `lightning-indexer`, `topk-select`, `cout-lineaire`, `adoption`.
