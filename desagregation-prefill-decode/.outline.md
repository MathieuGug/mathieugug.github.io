# Outline — Désagrégation prefill/decode

## Thèse centrale
La désagrégation prefill/decode sépare les deux phases de l'inférence LLM — l'une *compute-bound* et parallèle (prefill), l'autre *memory-bandwidth-bound* et séquentielle (decode) — sur des pools matériels distincts pour supprimer l'interférence qui plafonne le **goodput** (débit utile sous SLO). Entre 2024 et 2026 elle est passée du papier de recherche à l'architecture par défaut du serving à grande échelle ; mais elle n'est rentable qu'au-dessus d'un seuil de charge et de longueur de contexte — en dessous, le prefill découpé en colocation (Sarathi-Serve) reste compétitif.

## Plan (8 sections, ~6500 mots, 7 schémas)

1. **Synthèse exécutive** — 4-5 findings. Le pivot goodput. 7,4× (DistServe) / 2,35× (Splitwise) / 59-498% (Mooncake). Le seuil de rentabilité.
2. **Deux phases, deux régimes** — prefill compute-bound (parallèle sur tous les tokens du prompt, sature les FLOPs, écrit le KV cache) vs decode memory-bound (un token à la fois, lit/étend le KV cache, sous-utilise le compute). SCHEMA-01.
3. **L'interférence en colocation** — batching hybride : un prefill long bloque les decodes en cours (head-of-line blocking), pic de TBT jusqu'à 28,3× (Sarathi). Le tradeoff TTFT vs TBT. SCHEMA-02.
4. **Throughput n'est pas goodput** — pourquoi le débit brut ment ; goodput = req/s respectant TOUS les SLO (TTFT + TBT). Le knee se déplace. SCHEMA-03.
5. **L'alternative non-désagrégée : le prefill découpé** — Sarathi-Serve chunked prefill, stall-free scheduling, 2,6-5,6× capacité. Le baseline à battre, pas un homme de paille.
6. **L'architecture désagrégée de référence** — routeur KV-aware → pool prefill (peu de GPU haut de gamme) → fabric de transfert → pool decode (plus de GPU, moins chers) → KV store global. SLO planner. SCHEMA-04.
7. **Le nerf de la guerre : le transfert du KV cache** — bande passante, transfert layer-by-layer pipeliné (recouvre le compute), NIXL/RDMA/NVLink, le mur de bande passante, prefix cache global (Mooncake Store). SCHEMA-05.
8. **Cartographie 2024-2026 + quand désagréger** — DistServe, Splitwise, Mooncake, Dynamo, vLLM+LMCache, SGLang, TensorRT-LLM. SCHEMA-06 (matrice). Décision : charge × tension SLO. SCHEMA-07 (2×2). Horizon : MoE/expert-parallelism, reasoning models, KVCache-as-a-service.
9. **Sources**

## Schémas (7) + zones cliquables

- **SCHEMA-01** (square) — *Deux phases, deux régimes matériels*. Colonnes prefill vs decode + objet KV cache partagé + roofline compute/mémoire.
  Régions : `prefill-phase`, `decode-phase`, `kv-cache`, `roofline`.
- **SCHEMA-02** (landscape) — *L'interférence en colocation*. Timeline d'un batch : decodes en cours stallés par injection d'un prefill long (HOL blocking) ; pic TBT.
  Régions : `ongoing-decode`, `prefill-injection`, `hol-blocking`, `slo-violation`.
- **SCHEMA-03** (landscape) — *Throughput n'est pas goodput*. Deux courbes vs charge : throughput monte, goodput s'effondre au knee SLO ; knee décalé en désagrégé.
  Régions : `throughput-curve`, `goodput-curve`, `slo-knee`, `disagg-gain`.
- **SCHEMA-04** (landscape, Pattern A) — *Architecture désagrégée de référence*. Routeur → pool prefill → fabric transfert KV → pool decode → KV store + spine observabilité/SLO planner.
  Régions : `router`, `prefill-pool`, `kv-transfer`, `decode-pool`, `kv-store`, `slo-planner`.
- **SCHEMA-05** (landscape) — *Le transfert du KV cache*. Transfert layer-by-layer pipeliné (recouvre le compute prefill) ; backends NVLink/RDMA/NIXL ; mur de bande passante ; hit prefix cache.
  Régions : `layerwise-transfer`, `transport-backends`, `bandwidth-wall`, `prefix-cache-hit`.
- **SCHEMA-06** (landscape, Pattern C) — *Cartographie des systèmes 2024-2026*. Lignes : DistServe, Splitwise, Mooncake, Dynamo, vLLM+LMCache, SGLang, TensorRT-LLM. Colonnes : disagg natif, transport KV, KV cache global, ratio P:D dynamique, échelle prod, licence.
  Régions : `distserve`, `splitwise`, `mooncake`, `dynamo`, `vllm`, `sglang-trtllm`.
- **SCHEMA-07** (square, Pattern E) — *Quand désagréger — 2×2*. Axes : charge & longueur de contexte (x) × tension SLO / divergence TTFT-TBT (y). Quadrants : colocation simple, chunked prefill, désagrégation, KVCache-centric. + horizon 2026-2028.
  Régions : `q-colocation`, `q-chunked`, `q-disagg`, `q-kvcentric`.
