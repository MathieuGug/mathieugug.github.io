# Outline — La désagrégation prefill/decode

## Thèse centrale
La désagrégation prefill/decode fait passer le service d'inférence d'un **problème d'ordonnancement temporel** (partager un GPU entre deux phases antagonistes) à un **problème d'architecture distribuée** (placer, transférer, router) : la *phase* devient l'unité de déploiement, le KV-cache devient un flux réseau, et le *goodput* sous double SLO remplace le débit brut comme métrique de vérité.

## Plan (8 sections)
1. **Deux phases, un conflit irréductible** — prefill compute-bound / decode memory-bound ; pourquoi le co-placement plafonne (interférence + parallélisme imposé identique aux deux phases) ; recap roofline + renvoi kv-cache. [SCHEMA-01]
2. **Goodput : la métrique qui change la donne** — le débit agrégé trompe ; DistServe introduit le goodput = requêtes/s respectant *à la fois* TTFT (prefill) et TPOT (decode) ; deux SLO antagonistes sur un même matériel. [SCHEMA-02]
3. **Anatomie d'un système désagrégé** — routeur/orchestrateur → pool prefill → transfert KV → pool decode → pool KV global. Le contrat. [SCHEMA-03]
4. **Le placement par phase : parallélisme découplé** — chaque phase choisit son parallélisme (TP/PP/EP) et son ratio de nœuds ; DistServe co-optimise ; DeepSeek-V3 prefill (EP32) vs decode (EP320) ; ratio prefill:decode dynamique. [SCHEMA-04]
5. **Le coût central : transférer le KV-cache** — budget de bande passante, RDMA/NVLink/NIXL, recouvrement couche-par-couche, quand le lien annule le gain ; la localité comme contrainte de placement. [SCHEMA-05]
6. **Désagrégation contre chunked prefill : deux écoles** — Sarathi-Serve (stall-free, zéro transfert) vs DistServe/Splitwise (séparer physiquement) ; charges où chacune gagne ; convergence. [SCHEMA-06]
7. **La production : qui désagrège, et comment** — Mooncake/Kimi, Splitwise/Azure, DeepSeek-V3, frameworks (NVIDIA Dynamo, llm-d, vLLM/SGLang natif). [SCHEMA-07]
8. **Trajectoires 2026-2028** — désagrégation par défaut ; désagrégation à N phases (EP désagrégé) ; KV-as-a-service cross-request ; placement hiérarchique ; observabilité du transfert (gen_ai.*).
