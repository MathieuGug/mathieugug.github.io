# Outline — Benchmarks agentiques contestés

## Thèse centrale

Le score d'un agent sur SWE-bench, GAIA ou OSWorld n'est plus une mesure : c'est un produit. Quand cinq ingénieurs travaillent à temps plein pour gagner trois points de leaderboard, le chiffre raconte d'abord l'effort d'optimisation, pas la capacité.

## Plan — 7 sections (≈6 500 mots) + 6 schémas

1. **L'écart qui s'élargit** (≈700 mots) — Un agent qui annonce 70 % sur SWE-bench n'a pas 70 % de chances de réparer votre bug. Pourquoi cet écart est structurel, pas transitoire. Cadrage : le score comme proxy défaillant.

2. **Anatomie de SWE-bench** (≈900 mots) — Princeton 2024, 2 294 issues filtrées sur 12 repos Python, harnais de test, scoring binaire. Le pivot SWE-bench Verified (OpenAI, août 2024) et le doute qu'il a légitimé. **SCHEMA-01** — architecture du pipeline. *Lead* : « SWE-bench est devenu le réflexe. Voyons pourquoi le réflexe n'a jamais été innocent. »

3. **Les quatre vecteurs de fuite** (≈1 000 mots) — Taxonomie de la contamination : (i) chevauchement train/test via dates de cutoff, (ii) fuite par version-tag (issue résolue dans `pandas==1.3.5` → modèle apprend la version magique), (iii) gaming du harnais (Magic.dev, Cognition : retries, formatage, agent custom), (iv) leakage de prompt (instructions au modèle qui transportent des hints). **SCHEMA-02** — taxonomie 2×2. *Lead* : « Quatre fuites, quatre cultures différentes, un seul effet : décorréler le score de la capacité réelle. »

4. **GAIA, OSWorld, τ-bench : même pathologie, déclinée** (≈900 mots) — La saturation rapide de GAIA Level 1, l'écart simulation/réalité d'OSWorld, le user-simulator de τ-bench (Sierra) qui apprend lui aussi à coopérer. **SCHEMA-03** — trajectoire des scores 2024-2026 (multi-modèles). *Lead* : « Trois benchmarks, trois architectures, une même courbe en sablier. »

5. **Le score est un produit** (≈1 100 mots) — La "benchmark team" comme métier. Anthropic, Magic.dev, Cognition, OpenAI : pourquoi chaque labo a maintenant 3-5 ingénieurs dédiés au seul SWE-bench. Comment l'optimisation s'industrialise : harness custom, RL ciblé, retries, sélection de checkpoints, prompts confidentiels. La trajectoire ARC-AGI comme contre-exemple (Chollet). **SCHEMA-04** — chaîne d'optimisation industrielle. *Lead* : « Goodhart's law en accéléré : quand la mesure devient cible, elle cesse d'être mesure. »

6. **Le contre-mouvement : benchmarks vivants** (≈900 mots) — SWE-bench Live (Princeton 2025, rolling test set basé sur les commits postérieurs au cutoff), SWE-Lancer (paiement réel sur Upwork), CORE-Bench (reproductibilité scientifique), MLE-bench (Kaggle live). Les compromis : coût d'évaluation, comparabilité, biais de sélection. **SCHEMA-05** — matrice comparative. *Lead* : « Si le benchmark vieillit dès qu'il est publié, il faut qu'il vieillisse avec le modèle. »

7. **Mesurer pour quoi faire ?** (≈800 mots) — Ce que la mesure ne dit pas : généralisation hors-distribution, sécurité, coût latent (compute, tokens), comportement multi-tour, capacité à demander de l'aide. Framework 2×2 valeur/risque. Recommandation : ne jamais acheter un agent sur un seul leaderboard ; toujours doubler par une éval interne ad hoc. **SCHEMA-06** — framework décision. *Lead* : « Le bon usage d'un benchmark est diagnostique, pas évaluatif. »

## Sources — 12 sources tier-1

1. **Jimenez et al. (2024)** — *SWE-bench: Can Language Models Resolve Real-World GitHub Issues?* ICLR/NeurIPS 2024. Le papier fondateur. https://arxiv.org/abs/2310.06770
2. **OpenAI (2024)** — *Introducing SWE-bench Verified*. Blog officiel, août 2024. https://openai.com/index/introducing-swe-bench-verified/ — Sous-ensemble human-verified de 500 issues, légitime le doute sur le benchmark original.
3. **Mialon, Fourrier et al. (2023)** — *GAIA: a benchmark for General AI Assistants*. Meta AI + HuggingFace. https://arxiv.org/abs/2311.12983 — 466 tâches, 3 niveaux, originalement très difficile.
4. **Xie et al. (2024)** — *OSWorld: Benchmarking Multimodal Agents for Open-Ended Tasks in Real Computer Environments*. NeurIPS 2024. https://arxiv.org/abs/2404.07972 — 369 vraies tâches sur OS.
5. **Yao, Shinn et al. (2024)** — *τ-bench: A Benchmark for Tool-Agent-User Interaction*. Sierra. https://arxiv.org/abs/2406.12045 — User-simulator scriptable.
6. **Sainz, Campos et al. (2023)** — *NLP Evaluation in Trouble: On the Need to Measure LLM Data Contamination for each Benchmark*. EMNLP Findings 2023. https://arxiv.org/abs/2310.18018 — Le papier de référence sur la contamination.
7. **Roberts, Thakur et al. (2024)** — *Data Contamination Through the Lens of Time*. https://arxiv.org/abs/2310.10628 — Démonstration empirique du leakage temporel sur code benchmarks.
8. **Magar & Schwartz (2022)** — *Data Contamination: From Memorization to Exploitation*. ACL 2022. https://arxiv.org/abs/2203.08242 — Distinction memorization vs exploitation.
9. **Chollet (2024)** — *ARC Prize 2024: Technical Report*. https://arxiv.org/abs/2412.04604 — ARC-AGI comme contre-exemple méthodologique.
10. **Aleithan et al. (2024)** — *SWE-Bench+: Enhanced Coding Benchmark for LLMs*. https://arxiv.org/abs/2410.06992 — Audit des solution leakages dans SWE-bench original.
11. **Wang et al. (2025)** — *SWE-bench Live: A Continuously Updated Benchmark*. Princeton. (Rolling test set.)
12. **OpenAI (2024)** — *MLE-bench: Evaluating Machine Learning Agents on Machine Learning Engineering*. https://arxiv.org/abs/2410.07095 — 75 compétitions Kaggle, design anti-contamination.

## 6 schémas — briefs

- **SCHEMA-01** — *Anatomie de SWE-bench* — landscape architecture. Pipeline : 12 repos GitHub → 2 294 issues filtrées → harnais Docker → scoring binaire. Régions : collection, filtering, harness, scoring, leaderboard. **4 régions**.
- **SCHEMA-02** — *Quatre vecteurs de contamination* — 2×2 square. Axes : explicite/implicite × données/protocole. Quadrants : train-test overlap, version-tag leakage, harness gaming, prompt leakage. **4 régions**.
- **SCHEMA-03** — *Trajectoire des scores SWE-bench Verified 2024-2026* — timeline landscape. Y : score 0-100 %. X : Q1 2024 → Q2 2026. Lignes : Claude (Opus 3 → 4.7), GPT (4 → 5), agents spécialisés (Magic, Cognition Devin). Annotations sur SWE-bench Verified release, SWE-bench Live release. **4 régions** (points d'inflexion).
- **SCHEMA-04** — *Chaîne d'optimisation industrielle d'un score benchmark* — landscape flow. Étapes : sélection checkpoint, harness custom, prompt engineering, RL fine-tuning ciblé, retries N=8, sélection vote majoritaire, publication leaderboard. **5 régions**.
- **SCHEMA-05** — *Comparatif des nouvelles approches benchmark-vivantes* — landscape matrix. Lignes : SWE-bench Live, SWE-Lancer, CORE-Bench, MLE-bench, ARC-AGI 2. Colonnes : Mise à jour, Coût d'éval, Anti-contamination, Comparabilité, Biais. Indicateurs : ●/◐/○. **5 régions** (une par ligne).
- **SCHEMA-06** — *Framework décision : ce que la mesure dit, ce qu'elle ne dira jamais* — 2×2 square. Axes : contrôlé ↔ écologique × ponctuel ↔ longitudinal. 4 quadrants : benchmark public / éval interne / canary production / chaos test. **4 régions**.
